Hugo + github.io를 이용하여 무료 호스팅 블로그를 만들어보자. (2)
  • Sep 14, 2019
  • 4 minutes read

Hugo Framework를 이용해서 Github.io에 호스팅을 목표로 차례대로 작성해보도록 하겠습니다. 이번 저는 orianna-zzo/AllinOne 테마를 적용하는 과정을 이번 블로그에서 담아 보려고 하였습니다. 그렇다 보니 다른 테마를 적용하시는 분은 과정이 다른 부분도 있을 것이라고 생각됩니다. 테마 적용부터는 참고 정도로 봐주시면 좋을 것 같습니다.

1. Hugo를 설치해보자

Hugo를 설치하기 위해서는 두가지 방법이 있습니다.
1. 미리 컴파일 된 바이너리 파일
2. 설치 툴을 이용한 설치

위 두 방법중 하나의 방법을 선택하여 Hugo를 설치하여 주세요.

1.1 미리 컴파일 된 바이너리 파일

Hugo는 미리 Hugo Release에서 바이너리파일들을 제공합니다.
각 환경에서 바이너리 파일을 환경변수에서 등록하여 사용할 수 있습니다.

  • MacOS
  • Windows
  • Linux
  • OpenBSD
  • FreeBSD

1.2 설치 툴을 이용한 설치

각 환경에 맞는 설치툴을 이용하여 간단한 명령어로 Hugo를 설치 할 수 있습니다.

  • MacOS

    $brew install hugo

  • Linux
    LinuxBrew 패키지 매니저를 이용하여 설치 할 수 있습니다.

    $brew install hugo

  • Windows
    chocolatey 패키지 매니저를 이용하여 설치 할 수 있습니다.

    $choco install hugo -confirm

    또는, scoop 패키지 매니저를 이용하여 설치 할 수 있습니다.

    $scoop install hugo
더 많은 설치방법을 자세하게 확인하고 싶다면 https://gohugo.io/getting-started/installing/ 를 참고하시면 됩니다.

2. Hugo 프로젝트 만들기

프로젝트를 만들고자 하는 위치에 ProjectName 명으로 프로젝트를 생성합니다.

$hugo new site <ProjectName>

3. Hugo 테마 다운하기

기본적으로 Hugo에서는 많은 테마들을 지원하고 있습니다. 마음에 드는 Hugo 테마 에서 원하는 테마를 찾아보시면, “Demo”버튼을 통해서 컨셉을 확인할 수 있으며 “Download”버튼을 통해 들어가보면 해당 테마의 GitHub으로 이동하게 됩니다. “Clone or Download” 버튼을 누르면 Url을 복사합니다.

$cd <ProjectName>/theme
$git clone <GitHubSSHKey> // GitHubSSHKey 위치에 테마 URL을 복사

4. 테마 적용하기

다운 받은 테마를 적용하기 위해서는 config.toml 파일을 수정해야 적용할 수 있습니다. 편집 Editor를 이용하여 해당 부분을 수정합니다.

theme = "AllinOne" // themes 중 AllinOne 테마를 적용

위 형식으로 테마를 적용하면 되지만 AllinOne 테마는 위처럼 테마만 적용한다고 정상동작 하지 않습니다. themes/AllinOne/exampleSite을 참고해서 config.toml파일과 content 디렉토리 구조를 구성 해야지만 제대로 동작한다는 것을 명심해주시면 되겠습니다.

5. 포스트 생성하기

프로젝트 위치에서

hugo new <PostName.md> // md파일 형식으로 추가

명령어를 수행하게 되면 contents/PostName.md 가 생성됩니다.

6. 빌드 및 Git Repository 구성

Hugo는 jekyll과 다르게 github에서 자동으로 빌드해주는 것이 아니라 내가 빌드한 파일의 산출물을 <user-id>.github.io 저장소에 적용해야 정상적으로 호스팅이 됩니다. 그러므로 2개의 Git Repository가 필요합니다.

  • Project Repository
  • <user-id>.github.io Repository

Github에서 위 2개의 Repository를 생성합니다.

아래 과정으로 Git Repository를 구성하였습니다.

  1. 프로젝트에 public 디렉토리가 있는지 확인하고 만약 있다면 제거 해줄 수 있도록 합니다.

  2. 프로젝트 위치에서 아래 명령어를 수행합니다.

    $git init
$git remote add origin <프로젝트 Url>
$git submodule add -b master <github.io Url> public
$hugo -t AllinOne // 테마명으로 빌드를 해야합니다.

    위 명령어는 Clone or download 버튼을 누르면 URL을 확인할 수 있고 복사할 수 있습니다.

  3. 커밋 후 push 하여 github에 반영합니다.

    $cd public
$git add .
$git commit -m "github.io commit"
$git push origin master
$cd ..
$git add .
$git commit -m "first commit"
$git push origin master

    변경 이력이 있을 때에 위 Commit&Push를 하면 Github Repository에 적용됩니다.

https://<user-id>.github.io로 들어가보시면 작업하신 Hugo 테마 템플릿이 적용되신 것을 확인하실수 있습니다.

7. Hugo 서버를 실행해보기

사이트를 올리기 전에 간단한 명령어를 통해서 생성한 Hugo 사이트를 실행해 볼 수 있습니다.

$hugo server -D

명령어를 수행하면 로컬 영역에서 실행해 볼 수 있는데 http://localhost:1313에 접속한다. 접속해보면 블로그의 실제 모습을 확인 해 볼수 있으며 수정할 때마다 Live Reloading되어 실시간으로 변경 되는 블로그의 모습을 확인 할 수 있다.

8.

이번 블로깅을 통해서 제 개발 블로그를 만들게 되었고 아직은 부족한 모습이지만 앞으로 다양한 기능들을 추가 해볼 생각입니다. 앞으로도 Hugo를 이용해서 Github.io 호스팅해보기를 지속해보도록 하겠습니다. 이글이 다른분에게 도움이되었으면 합니다. 감사합니다.
