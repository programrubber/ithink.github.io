Hugo + github.io를 이용하여 무료 호스팅 블로그를 만들어보자. (1)
  • Sep 14, 2019
  • 2 minutes read

1. Hugo를 설치해보자

Hugo를 설치하기 위해서는 두가지 방법이 있습니다.
1. 미리 컴파일 된 바이너리 파일
2. 설치 툴을 이용한 설치

1.1 미리 컴파일 된 바이너리 파일

Hugo는 미리 Hugo Release(Link)에서 바이너리파일들을 제공합니다.
각 환경에서 바이너리 파일을 환경변수에서 등록하여 사용할 수 있습니다.

  • MacOS
  • Windows
  • Linux
  • OpenBSD
  • FreeBSD

1.2 설치 툴을 이용한 설치

각 환경에 맞는 설치툴을 이용하여 간단한 명령어로 Hugo를 설치 할 수 있습니다.

  • MacOS

    $brew install hugo

  • Linux
    LinuxBrew(Link) 패키지 매니저를 이용하여 설치 할 수 있습니다.

    $brew install hugo

  • Windows
    chocolatey(Link) 패키지 매니저를 이용하여 설치 할 수 있습니다.

    $choco install hugo -confirm

    또는, scoop(Link) 패키지 매니저를 이용하여 설치 할 수 있습니다.

    $scoop install hugo
더 많은 설치방법을 자세하게 확인하고 싶다면 https://gohugo.io/getting-started/installing/ 를 참고하시면 됩니다.

2. Hugo 프로젝트 만들기

프로젝트를 만들고자 하는 위치에 ProjectName 명으로 프로젝트를 생성합니다.

$hugo new site <ProjectName>

3. Hugo 테마 다운하기

기본적으로 Hugo에서는 많은 테마들을 지원하고 있습니다. 마음에 드는 Hugo 테마(Link) 에서 원하는 테마를 찾아보시면, 원하는 테마는 “Demo”를 통해서 컨셉을 확인할 수 있으며 원하는 테마를 “Download”를 통해 들어가보면 해당 테마의 GitHub으로 이동하게 됩니다. “Clone or Download” 버튼을 누르면 Url을 복사합니다.

$cd <ProjectName>/theme
$git clone <GitHubSSHKey> // GitHubSSHKey 위치에 테마 URL을 복사

4. 테마 적용하기

다운 받은 테마를 적용하기 위해서는 config.toml 파일을 수정해야 적용할 수 있습니다. 편집 Editor를 이용하여 해당 부분을 수정합니다.

theme = "AllinOne"

위 형식으로 테마가 적용되지 않는다면, 보통 플러그인 안에 config.toml을 예제로 가지고 있는 경우가 있으므로 자신의 프로젝트에 Copy & Paste 하도록 합니다..

5. 포스트 생성하기

프로젝트 위치에서

hugo new <PostName.md> // md파일 형식으로 추가

명령어를 수행하게 되면 contents/PostName.md 가 생성됩니다.

6. Hugo 서버를 실행해보기

사이트를 올리기 전에 간단한 명령어를 통해서 생성한 Hugo 사이트를 실행해 볼 수 있습니다.

$hugo server -D

명령어를 수행하면 로컬 영역에서 실행해 볼 수 있는데 http://localhost:1313에 접속한다. 접속해보면 블로그의 실제 모습을 확인 해 볼수 있으며 수정할 때마다 Live Reloading되어 실시간으로 변경 되는 블로그의 모습을 확인 할 수 있다.
