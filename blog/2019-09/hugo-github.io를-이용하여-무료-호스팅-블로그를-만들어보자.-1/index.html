Hugo + github.io를 이용하여 무료 호스팅 블로그를 만들어보자. (1)
  • Sep 14, 2019
  • 4 minutes read

문서화작업을 할때에 사용할 블로그 플랫폼을 알아보던 중에 보통 Github.io 라는 URL이 어느순간부터 보이기 시작하였고 알아본 결과 많은 개발자들이 개발 블로그용으로 많이 사용되고 있다는 것을 알게 되었습니다. github.io는 왜 개발자들에게 인기가 많고 많이들 사용하고 있는 것일까요?

1. Github Page

Github.io는 Github Pages의 URL형태 입니다.
Github 저장소의 내용을 직접 웹 페이지를 통해서 보여 줄 수 있으며 무료로 보안적인 요소가 적용된 HTTPS 웹 서버를 구축할 수 있습니다.
username.github.io의 이름을 가진 저장소를 만든다면 http://username.github.io 웹사이트의 기본도메인이 됩니다.
Github Pages 공식사이트에 가본다면 좀더 자세한 설명을 확인해 볼 수 있습니다.

username이 아닐 경우 정상동작 하지 않으므로 꼭 사용자 계정명을 사용해야 합니다.

2. Static site generator

정적 사이트 생성기(Static site generator, SSG)라 부를 수 있습니다. 많은 웹 사이트들이 동적이지만 여전히 정적 사이트 들도 사용되고 있습니다.
실제로 마크다운형식으로 작성하고 HTML형식으로 빌드 하는 정적 사이트의 사용이 증가 하고 있어 유행은 돌고 돈다는 생각이 들곤 합니다.
개발적인 측면에서도 동적인 사이트에 비한다면 개발이 쉽다는 여러장점이 있으며 아래와 같은 이유로 많이 사용되고 있습니다.

  • 동적인 제품에 비해 매우 빠릅니다.
  • 적은 유지보수
  • 보안 수준이 높습니다.
  • 포트폴리오와 같은 간단한 웹사이트를 구축하는데에 적합합니다.

관리도 힘들고 새 글을 쓰거나 수정할 때도 글을 다운로드 받아 편집하고 업로드까지 수동으로 해야하는 단점들을 가지고 있어 확장성면에서는 한계가 있습니다.

Hugo와 Jekyll은 인기가 많은 정적 사이트 생성기로 유명합니다. 저는 이 둘 중 어떤 프레임워크를 사용해볼지 비교해 보도록 하겠습니다.

3. Jekyll?

2009년 Tom Preston-Werner가 출시 한 Jekyll은 가장 오래된 static site general 입니다.
Jekeyll은 Ruby언어로 작성 되어 전세계적으로 사용 되고 있습니다. Github과의 제휴를 하게되어 Tom Preston-Werner는 Github의 공동설립자가 되어 Jekyll은 Open Source Comunity로 부터 많은 관심과 기여를 받게 됩니다.

Jekyll은 HTTP 서버와 함께 사용되며 일반적으로 블로그, 포트폴리오를 만드는 데에 사용되고 있습니다.

Jekyll은 페이지 컨텐츠에 Markdown을 사용하지만 Liquid templating 언어를 사용하여 페이지를 처리하고 있습니다.
Sass도 사용되고 있는데, 이는 CSS 전처리기를 좋아하는 개발자들에게 큰 영향을 주게 됩니다.

Jekyll에 기여하고 있는 많은 개발자들이 있기 때문에 많은 플러그인을 가지고 있습니다.

  • 장점
    • Jekyll을 설치하고 배포하는 과정은 간단합니다.
    • 방대한 개발자 커뮤니티를 갖추고 있어 도움을 받을 사람들이 많습니다.
    • 배우기 쉬운 Liquid templating 언어를 사용합니다.
    • Jekyll은 검색 엔진 최적화 SEO에 적합합니다.
    • 다양한 플러그인이 제공됩니다.
  • 단점
    • Jekyll은 Ruby 개발환경이 필요하기 때문에 Windows 사용자를 위한 설정이 어려울 수 있습니다.
    • Jekyll은 사이트의 내용이 방대해 질 수록 속도가 느려질 수 있습니다.

4. Hugo?

Hugo는 웹 사이트를 빌드하는데 빠른 성능을 자랑하는 프레임워크 라고 합니다. Hugo는 Go언어를 사용하고 있으며 2014년도에 출시되었습니다. Go언어는 빠른 언어 중 하나이며 Hugo는 이에 영향을 받아 성능 상에 이점이 있습니다. Hugo는 속도 향상의 필요성에 의해서 특별히 제작되었으며 ms 단위의 시간으로 빌드를 빠르게 처리합니다. Go 템플릿을 기반으로 한 템플릿을 사용하며 HTTP 서버와 함께 제공되므로 완전한 패키지라고 할 수 있습니다.

성능과 다양한 내장기능으로 인해 Hugo는 블로그와 문서를 생성하는데 사용되고 있습니다.

  • 장점

    • 다른 어떤 정적 프레임워크 보다 빠릅니다.
    • 많은 내장 기능을 사용할 수 있어 타사의 플러그인이 필요하지 않습니다.
    • Hugo는 설치가 쉽고 Windows, Linux, Mac 등등 다양한 OS 플랫폼을 지원합니다.
    • 쉽게 템플릿을 사용할 수 있습니다.

  • 단점

    • Go언어를 배우는 것은 어려울 수 있습니다.

    • 데이터 파일 형식으로 XML을 지원하지 않습니다. 그러나 YAML, JSON, CSV는 지원합니다. `

두 프레임워크의 장단점이 있으며 모두 훌륭한 프레임워크라 생각합니다. 하지만 제가 Hugo Framework를 Study하여 Blog를 만드는 이유는 Go언어를 좀더 공부 하고 싶다는 부분과 무엇보다도 Jekyll 보다 빠르다는 점입니다. 실제로 Hugo로 Blog를 만들면서 빌드가 1초도 안되어 완료된다는 점은 정말 신세계라고 할 수 있었습니다. 이번 블로그를 작성하며 Go언어에 좀더 친숙해지고 파생적으로 어떤 활용성이 있는 언어인지도 확인해볼 생각입니다. 또 Hugo는 다양한 OS 플랫폼에 바이너리를 직접 빌드하여 릴리즈 하고 있다는 장점이 있어 어디에서나 개발을 쉽게 할 수 있다는 점도 크게 영향이 있었습니다.

다음 번에는 Hugo를 설치 및 테마적용, Github.io에 나만의 블로그를 만드는 과정을 담아 보도록 하겠습니다.
