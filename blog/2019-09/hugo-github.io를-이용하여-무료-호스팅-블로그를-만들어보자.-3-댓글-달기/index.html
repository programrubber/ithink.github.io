Hugo + github.io를 이용하여 무료 호스팅 블로그를 만들어보자. (3) - 댓글 달기
  • Sep 21, 2019
  • 4 minutes read
`orianna-zzo/AllinOne` 테마에 `utterences` 댓글 생성기를 구축하는 과정을 담았습니다.

이번 포스트에서는 Hugo + github.io를 이용하여 무료 호스팅 블로그를 만들어보자. (2)에서 블로그를 Github.io에 호스팅을 하는 과정을 진행하였습니다. 이번 과정에서는 포스팅에 다른 사람들의 댓글을 달 수 있게 해주는 utterences를 추가하려고 합니다. utterences는 블로그에 댓글을 추가 할 수 있도록 API를 지원해주고 있으며 기능들을 구현하지 않더라도 간단하게 구현할 수 있는 장점을 가지고 있습니다.

1. utterances?

utterances는 Github의 이슈기능으로 블로그에 댓글을 달 수 있는 있게 해주는 블로그 댓글 생성기입니다.
utterances는 왜 사용되고 있을까요?

  • Open source
  • 트랙킹, 광고를 하지 않으며, 항상 무료입니다.
  • Github Issue안에 모든 데이터가 저장됩니다.
  • GitHub를 지원하는 CSS 툴킷 인 Primer로 스타일이 지정되었습니다.
  • 다크 테마를 지원합니다.
  • 가볍습니다. Vanilla TypeScript 입니다. 폰트를 다운 할 필요가 없고, JS프레임워크와 polyfill이 없습니다.

서비스 회사의 DB를 이용하는 disqus와는 다르게 Github Issue를 사용하고 있어 DB가 불필요하다는 장점을 가지고 있습니다. 또한 무료라는 점이 가장 마음에 듭니다. utterrances는 하나의 포스팅에 하나의 Github Issue를 매핑하는 구조를 가지고 있습니다. Github Repository Issue 에서 포스팅 들의 댓글 들을 관리 할 수 있으며 블로그에서 뿐 아니라 Github Issue에서 피드백을 해줄 수 있다는 매력을 가지고 있어 댓글 관리적인 면에서 편하다고 할 수 있습니다.

포스트의 댓글 부분은 iframe으로 로딩되며 Page의 url, pathname(or title)로 issue search API 를 통해 해당 이슈에 달린 댓글을 가져와 렌더링하여 보여주게됩니다.

2. utterances install

utterances를 적용하고자 하는 Repository에 Issue에 접근 할 수 있도록 github에 설치(권한 획득) 하는 과정이 필요합니다.

  1. github app설치 페이지에서 utterances를 설치합니다.
  2. Only select repositories 를 선택 합니다.
  3. github repository 를 선택해주면 된다.
    • 저는 github계정.github.io를 선택하였습니다.
    • 혹시라도 다른 Repository를 선택하실 경우, private 저장소는 안된다는 점을 참고하여 주시길 바랍니다.

3. 설정 스크립트 만들기

Utterances를 블로그에 반영하기 위해서는 특정 스크립트를 추가해줘야합니다.
스크립트를 생성하기 위해서는 utterances의 configuration에서 아래 항목을 작성하면 특정 스크립트를 생성하여 줍니다.

3-1. 저장소 선택

utterances는 포스트글 하나 Github Issue를 연동하는 구조입니다.
Iusse와 연결할 Respository를 입력합니다.

3-2. 포스트와 github issue 매핑하기

연동 하는 방법으로는 6가지의 방법 중 하나를 선택해야 합니다.

  1. Issue title contains page pathname
    • 게시물의 URL 경로 위치를 검색하고, Utterances 사용자가 처음 댓글을 달 때에 이슈를 생성합니다.
  2. Issue title contains page URL
    • 게시물의 URL을 가진 이슈를 검색하고, Utterances 사용자가 처음 댓글을 달 때에 이슈를 생성합니다.
  3. Issue title contains page title
    • 게시물의 제목으로 위치를 검색하고, Utterances 사용자가 처음 댓글을 달 때에 이슈를 생성합니다.
  4. Issue title contains page og:title
    • 게시물 페이지에 Open Graph title meta를 검색하고, Utterances 사용자가 처음 댓글을 달 때에 이슈를 생성합니다.
  5. Specific issue number
    • 게시물의 이슈 번호를 검색하고, Utterances 사용자가 처음 댓글을 달 때에 이슈를 생성합니다.
  6. Issue title contains specific term
    • 이슈 제목에 특정 단어가 있는지 검색하고, Utterances 사용자가 처음 댓글을 달 때에 이슈를 생성합니다.

저는 파일명을 변경하지 않을 계획이므로 세 번째 방법인 Issue title contains page title을 선택하였습니다. 여러분들도 블로그 구조에 따라서 위 방법 중 하나를 선택하도록 합니다.

3-3. 테마 적용하기

iframe으로 로딩하는 형식이기 때문에 블로그 Css테마와 다를 수 있다는 문제점을 가지고 있는데요.
utterances는 이를 보완하기 위하여 테마를 지원하고 있습니다. 원하는 테마를 선택합니다.

3-4. 스크립트 블로그에 적용하기

<!-- Comment -->
  <script src="https://utteranc.es/client.js"
    repo="[Repository]"
    issue-term="title"
    theme="github-light"
    crossorigin="anonymous"
    async>
  </script>
<!-- /.Comment -->

자신의 포스트가 입력되는 부분의 특정 스크립트를 추가합니다. 저는 제가 사용하는 테마인 orianna-zzo/AllinOne에서는 themes/llinOne/layouts/_default/single.html파일 위치에 특정 스크립트를 추가하였습니다. 다른 테마를 사용하시는 분들은 참고하여 주시길 바랍니다.

위와 같이 포스트 하단에 댓글을 달 수 있게 되었습니다.

4. 구성 후기

이번 포스팅을 통해서 댓글 달기 기능을 추가 하기 전에는 DB와 같은 요소들이 필요하다고 생각하고 있었습니다. 하지만 utterances는 그런 저의 생각을 바꿔주는 라이브러리 였습니다. utterances 쉽게 댓글 기능을 추가할 수 있을 수 있고 Github Issue에서 모든 포스트에 대한 댓글을 관리 할 수 있다는 분명한 매력이 있기 때문에 이제 시작하는 저보다도 블로그 생활을 즐기시는 분에게는 분명 도움이 되지 않을까 합니다.
이 글이 어느분에게는 도움이되었으면 합니다.
